# Story 4.1: Dashboard Overview & Stats

## Status: Ready for Dev

## Story

As a user,
I want to see a high-level summary of my spending,
So that I know my financial status at a glance.

## Acceptance Criteria

1. **AC1: Total Spending** - Display total spending for current month
2. **AC2: Receipt Count** - Display number of receipts uploaded
3. **AC3: Average/Receipt** - Display average spent per receipt
4. **AC4: Period Switcher** - Toggle between Week, Month, Year
5. **AC5: Comparison** - Show % change vs previous period

## Dev Notes
- **UI:** "Stats Grid" at top of Dashboard.
- **Backend:** SQL Aggregations (`SUM(total_amount)`, `COUNT(id)`).

# Story 4.2: Category Breakdown (Visuals)

## Status: Ready for Dev

## Story

As a user,
I want to see my spending split by category,
So that I know where my money is going.

## Acceptance Criteria

1. **AC1: Pie Chart** - Visual distribution of spending
2. **AC2: Category Colors** - Use the category's assigned color
3. **AC3: Tooltips** - Hovering shows Amount and Percentage
4. **AC4: Dynamic Updates** - Updates when Time Period changes (from Story 4.1)

## Dev Notes
- **Library:** `Recharts` (<PieChart>, <Cell>).
- **Data:** Group by Category JOIN.

# Story 4.3: Spending Trends

## Status: Ready for Dev

## Story

As a user,
I want to see my spending over time,
So that I can spot spikes or patterns.

## Acceptance Criteria

1. **AC1: Line/Bar Chart** - Spending over time (X-axis: Time, Y-axis: Amount)
2. **AC2: Granularity** - By Day (for Week view), By Week (for Month view)
3. **AC3: Responsive** - Readable on mobile

## Dev Notes
- **Library:** `Recharts` (<BarChart> or <AreaChart>).
- **Data:** `date_trunc` queries in Postgres.
